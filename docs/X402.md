# x402 for any app — Casper micropayments SDK

Charge for **anything served over HTTP** with one middleware on the server and
one wrapped `fetch` on the client. Payments settle on **Casper** through the
CSPR.cloud x402 facilitator as real CEP-18 token transfers. No WebRTC required —
that's just the first app built on this layer.

- Server: [`paymentMiddleware`](#server--gate-a-route) / [`X402Gate`](#framework-agnostic-gate)
- Client: [`wrapFetch`](#client--auto-pay)
- Runnable: [`examples/paid-api`](../examples/paid-api) — proven live on testnet
  (e.g. deploy [`3e04ec4e…`](https://testnet.cspr.live/deploy/3e04ec4e1ade22418747d7c36f0b5f71554c27d861180ea317bd2f48958465e7)).

## Install

```bash
npm i @nickthelegend69/webrtc-payment-sdk-core @nickthelegend69/webrtc-payment-rail-x402
```

## The flow

```
GET /thing                          → 402 + { accepts: [requirements] }
GET /thing  + X-PAYMENT: <payload>  → verify → settle on-chain → 200 (+ X-Payment-Tx)
```

The signed `PaymentPayload` rides in the `X-PAYMENT` request header (base64 JSON).
The 402 body carries the requirements the client must satisfy.

## Server — gate a route

```ts
import express from "express";
import { paymentMiddleware } from "@nickthelegend69/webrtc-payment-sdk-core";
import { CasperX402Rail } from "@nickthelegend69/webrtc-payment-rail-x402";

const rail = new CasperX402Rail({
  facilitatorUrl: "https://x402-facilitator.cspr.cloud",
  facilitatorApiKey: process.env.CSPR_CLOUD_API_KEY,   // server secret
  network: "casper:casper-test",
  tokenContractHash: process.env.CEP18_TOKEN_CONTRACT,
  token: { name: "Casper X402 Token", version: "1", decimals: "9", symbol: "X402" },
  providerAccountHash: process.env.PROVIDER_ACCOUNT_HASH,
});

const app = express();
app.get(
  "/premium",
  paymentMiddleware({
    rail,
    amount: "100000000",                 // 0.1 token (9 decimals)
    payTo: process.env.PROVIDER_ACCOUNT_HASH,
    asset: process.env.CEP18_TOKEN_CONTRACT,   // optional: pin the token
    network: "casper:casper-test",             // optional: pin the network
    // settle: false,                          // verify-only gate (no on-chain tx)
  }),
  (req, res) => res.json({ content: "🔓 unlocked" }),
);
```

`amount` can be a function of the request (`(req) => "..."`) for per-resource
pricing. On success the tx hash is set on the `X-Payment-Tx` response header.

### Framework-agnostic gate

No Express? Use `X402Gate` directly (Hono, Next API routes, plain `http`, …):

```ts
import { X402Gate, X402_PAYMENT_HEADER } from "@nickthelegend69/webrtc-payment-sdk-core";

const gate = new X402Gate({ rail, amount: "100000000", payTo, asset, network });

// in your handler:
const result = await gate.process(req.headers[X402_PAYMENT_HEADER]);
if (!result.ok) {
  res.writeHead(402, { "content-type": "application/json" });
  res.end(JSON.stringify({ x402Version: 2, accepts: [result.requirements], error: result.error }));
  return;
}
// result.txHash → settled; result.payer → who paid
serveContent(res);
```

## Client — auto-pay

```ts
import { wrapFetch } from "@nickthelegend69/webrtc-payment-sdk-core";
import { CasperX402Rail, makeEd25519SignFn, createBrowserSigner } from "@nickthelegend69/webrtc-payment-rail-x402";

const rail = new CasperX402Rail({
  facilitatorUrl: "https://x402-facilitator.cspr.cloud",
  network: "casper:casper-test",
  tokenContractHash: process.env.CEP18_TOKEN_CONTRACT,
  token: { name: "Casper X402 Token", version: "1", decimals: "9", symbol: "X402" },
  consumerAccountHash: walletAddress,
  consumerPublicKeyHex: publicKeyHex,
});

const signFn = createBrowserSigner();                 // browser wallet (CSPR.click)
// const signFn = makeEd25519SignFn(SEED_HEX);        // server/agent raw key (testnet)

const pay = wrapFetch(fetch, {
  rail,
  signFn,
  maxValue: "1000000000",                             // never auto-pay above 1 token
  onPayment: (req) => console.log("paying", req.amount),
});

const res = await pay("https://api.example.com/premium");   // the 402 is invisible
```

## Security model (built in)

The gate does **not** trust the client's stated price or payee. Before settling
it checks the signed requirements against your config:

- `payTo` must equal your account — a client can't redirect funds to itself.
- `amount` must be ≥ your price — no underpayment.
- `asset` / `network` must match if you pin them.

Then the facilitator verifies the EIP-712 signature, and only a valid payment is
settled. The `maxValue` cap on the client stops a malicious server from
over-charging.

## Robustness

- Facilitator calls have a **timeout** (default 20s) and **retry** transient 5xx
  (configurable via `timeoutMs` / `retries` on the rail).
- The rail validates config up front — a missing `token.name`/`version` throws a
  clear error instead of the facilitator's opaque
  `invalid_exact_casper_missing_token_name`.
- `X402Error.code` lets clients branch (`payment_exceeds_max`, `invalid_payment`, …).

## Notes

- One settle costs ~6 CSPR of gas on testnet (paid by the facilitator relayer).
  Tune price/segment length to your economics.
